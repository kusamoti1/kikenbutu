# 危険物法令ナレッジグラフAI — 完全技術監査レポート

**監査日:** 2026-03-04
**監査対象:** kikenbutsu_ai 全ソースコード

---

## 総合評価

| 観点 | 評価 |
|------|------|
| プロトタイプ | ○ 基本構造は整っている |
| 実用ツール | △ 重大なバグ修正が必要 |
| プロダクションレベル | × 検索機能・データ投入の根本的修正が必要 |

**現状レベル:** 初期プロトタイプ（構造設計は良好だが、主要機能が未接続）

---

## ① 重大な問題（5件）

1. **検索エンジンが機能していない** — `_embed()` が1次元ハッシュで意味的検索不可
2. **Streamlitアプリのインポートパスが壊れている** — `ModuleNotFoundError` で起動不可
3. **OCR処理がパイプラインに接続されていない** — データベースが空のまま
4. **equipment/standardsテーブルにデータ未投入** — 査察AI・NotebookLMエクスポートが機能しない
5. **データベース接続リーク** — SearchEngineに`close()`メソッドなし

## ② 潜在的バグ（7件）

1. 年代検索タブがボタンなしで毎回全件検索
2. PaddleOCR結果のnested list構造の未処理
3. 年代検出が最初のマッチのみ返す（複数年代未対応）
4. パイプライン再実行時のデータ重複
5. ナレッジグラフのノード名衝突
6. 法令条文パターンの不完全性（「第○条の○」未対応）
7. 改正理由抽出パターンが同一行のみ

## ③ パフォーマンス改善（4件）

1. FAISSインデックスの毎回再構築（Streamlitキャッシュ未使用）
2. 辞書補正のO(N×M)計算量
3. PDF変換400DPIの高メモリ消費
4. PaddleOCRの毎画像インスタンス化

## ④ セキュリティ改善（2件）

1. 依存パッケージのバージョン未固定
2. ファイル名のサニタイズ不足（低リスク）

## ⑤ NotebookLM制限評価

- ファイル数: 最大7ファイル（50制限に対して問題なし）
- ファイルサイズ: 現在テンプレートのみで空。修正後はサイズ監視が必要
- 設備×年代分割の検討を推奨

## ⑥ 初心者運用リスク（5件）

1. OCRテキストの手動配置が必要
2. PaddleOCRのインストール困難
3. エラーメッセージの不足（ログのみ）
4. パイプライン再実行時のデータ重複
5. バックアップ機構なし

## ⑦ スマホ環境問題（3件）

1. `layout="wide"` でモバイル表示に問題
2. パイプライン実行はPC必須
3. OCR関連ライブラリはモバイル非対応

## ⑧ 長期運用リスク（5件）

1. 検索がそもそも機能していない
2. SQLiteの同時アクセス制限
3. ナレッジグラフの肥大化
4. ログローテーションなし
5. スキーママイグレーション機構なし

## ⑨ 設計改善提案

1. 検索エンジン → SQLite FTS5に置換
2. OCR処理のパイプライン統合
3. Streamlitキャッシュの活用
4. 全テーブルへのデータ投入ロジック追加
5. 重複チェック・バックアップ機構の追加
